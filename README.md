# repo2xml

Convert a source code repository into a single, structured context document for LLM ingestion.

`repo2xml` walks a repository, applies `.gitignore` filtering (recursively, via a Git‑compatible scoped gitignore stack), emits a directory tree, and (optionally) embeds file contents with metadata in a deterministic format.

The internal architecture is pipeline‑based (Scan → Ingest → Serialize → Write). This makes it easier to add future output formats (JSON), API layers (gRPC/HTTP), and agent integrations, while keeping the current CLI behavior stable.

## Features

- **Project structure** as an XML tree (`<project_structure>` with `<dir>` / `<file>`).
- **File metadata**: relative path, extension(s), UTC mtime (optional), size (optional), symlink info.
- **Content modes**:
  - `full`: metadata + content (default)
  - `metadata`: metadata only (no content reads; not treated as "skipped")
  - `structure`: structure only

- **Robust Filtering**:
  - Respects `.gitignore` with correct scoping rules (patterns are applied relative to the directory containing the `.gitignore`).
  - Extra `--ignore` patterns (gitignore syntax).
  - **Force Include** (`--include`) to override ignore rules (patterns are automatically negated, i.e. “un‑ignore”).
  - **Hard excludes** (`--hard-exclude`, repeatable) – directory names that are never traversed or included (default: `.git`). Hard excludes cannot be overridden by `--include`.
  - Toggle `.gitignore` usage with `--gitignore` / `--no-gitignore`.

- **Smart Symlink Handling**:
  - Control symlink files behaviour via `--symlinks-files {follow|skip|as-link}` (default: `follow`).
  - `as-link` mode emits link metadata without reading content, saving tokens; broken symlinks are still included.
  - Control symlinked directory traversal with `--follow-symlinks-dirs` (default: disabled). When enabled, cycle protection is active.
  - Safe directory traversal with visited‑key tracking avoids infinite loops.

- **Fault Tolerance**: Read errors (permissions, encoding) do not crash the tool; they are reported as `<error>` tags within the output.

- **Binary handling**: skip, embed as base64, or store SHA‑256 hash.
  - `base64` and `hash` are computed in a streaming fashion (no full binary read into memory).
  - Configurable via `--binary` (default: `skip`).

- **Binary extension fast‑path** (configurable):
  - Case‑insensitive matching (PNG == png).
  - Can be disabled (`--no-ext-binary-detect`) or extended/trimmed with `--binary-ext-add` / `--binary-ext-remove`.

- **Formatting modes**: `compact` (default), `pretty` (indented with tabs), or `minify`.

- **Output**: File or stdout; optional gzip/zstd compression (`--compress`); clipboard support; dry‑run mode (`--dry-run`) that lists files without generating output; stats‑only mode that discards generated bytes.

- **Deterministic output options**:
  - `--no-timestamp` omits `generated_at_utc` for stable diffs.
  - `--no-mtime` omits `mtime_utc` attributes for stable diffs.
  - `--no-size` omits `size` attributes (determinism / privacy).
  - `--root-path-mode relative|redact` avoids leaking absolute paths.
  - `<root_path>` always uses POSIX separators (`/`) for reproducibility.

- **Newline normalization** (`--newline`): keep original line endings (`preserve`, default) or convert to LF (`lf`) for cleaner diffs and fewer tokens.

- **Built‑in safety**:
  - `--redact-secrets` redacts common secret‑like patterns (best‑effort) from embedded text.

## Quick Start

> Note: This CLI is implemented as a Typer app. To ensure arguments are parsed correctly, place options before the PATH.

Generate full XML for the current directory (default):

```bash
repo2xml -o context.xml
```

Run as a module:

```bash
python -m repo2xml -o context.xml .
```

Deterministic output (omit timestamp and redact root path):

```bash
repo2xml --no-timestamp --root-path-mode redact -o context.xml .
```

More deterministic output (omit timestamp + mtime + size, normalize newlines):

```bash
repo2xml --no-timestamp --no-mtime --no-size --root-path-mode redact --newline lf -o context.xml .
```

Emit only metadata for text files and treat symlinks as links (no content reads):

```bash
repo2xml --mode metadata --symlinks-files as-link -o context.xml .
```

Dry‑run: list files that would be processed (useful for testing filters):

```bash
repo2xml --dry-run --ignore "*.log" --include "!important.log"
```

Show version:

```bash
repo2xml --version
```

Compute stats only (discard output bytes):

```bash
repo2xml --stats-only --report .
```

## CLI Options

Show help:

```bash
repo2xml --help
```

### Output selection

- `-o, --output PATH` – Write output to file (default: `context.xml`). Ignored when `--stdout`, `--clipboard`, or `--stats-only` are used.
- `--stdout` – Write output to stdout.
- `--clipboard` – Copy output to system clipboard.
- `--stats-only` – Compute and print statistics, discard generated output.
- `--compress {none,gzip,zstd}` – Apply compression to the output stream (default: `none`).
- `--dry-run` – Do not write output; only list files that would be processed (respects all filtering options).

### Meta and determinism

- `--no-timestamp` – Omit `<generated_at_utc>` from the output meta block.
- `--no-mtime` – Omit `mtime_utc` file attributes.
- `--no-size` – Omit `size` file attributes (determinism / privacy).
- `--root-path-mode {absolute,relative,redact}` – How to emit `<root_path>`:
  - `absolute` (default): full resolved path (POSIX separators)
  - `relative`: relative path from current working directory when possible (POSIX separators)
  - `redact`: emit `<redacted>`

### Size limits

- `--max-size BYTES` – Maximum size in bytes for embedding **text content** and **base64 content** (when `--binary base64`). Larger files are emitted as skipped entries with a reason.  
  Binary hashing (`--binary hash`) is **not** subject to this limit.

### Decoding

- `--decode-errors {replace,strict}` – Text decoding policy:
  - `replace` (default): best‑effort decode with replacement characters
  - `strict`: fail on decode errors (emitted as error entries)
- `--newline {preserve,lf}` – Newline normalization:
  - `preserve` (default): keep original line endings
  - `lf`: convert CRLF/CR to LF (useful for deterministic output)

### Filtering

- `--gitignore` / `--no-gitignore` – Respect `.gitignore` files (default: enabled).
- `-i, --ignore PATTERN` – Additional ignore patterns (gitignore syntax). Repeatable.
- `--include PATTERN` – Additional include patterns (gitignore syntax). Repeatable. Patterns are treated as negations, overriding previous ignore rules.
- `--hard-exclude NAME` – Directory names to always exclude from traversal (repeatable). Default: `.git`. These are hard excludes and cannot be overridden by `--include`.

### Symlink handling

- `--follow-symlinks-dirs` / `--no-follow-symlinks-dirs` – Follow symlinked directories (default: disabled). When enabled, cycle protection is active.
- `--symlinks-files {follow,skip,as-link}` – How to handle symlink files:
  - `follow` (default): read the target (normal file behaviour)
  - `skip`: exclude symlink files entirely
  - `as-link`: emit metadata and link target without reading content

### Binary handling

- `--binary {skip,base64,hash}` – How to handle binary files (default: `skip`).
- `--ext-binary-detect` / `--no-ext-binary-detect` – Enable/disable binary extension fast‑path (default: enabled).
- `--binary-ext-add EXT` – Add extensions to treat as binary (repeatable). Case‑insensitive. Example: `--binary-ext-add .psd`
- `--binary-ext-remove EXT` – Remove extensions from the default binary fast‑path set (repeatable). Example: `--binary-ext-remove .pdf`

### Formatting

- `--formatting {compact,pretty,minify}` – Output formatting style (default: `compact`).
- `--mode {full,metadata,structure}` – High‑level output mode (default: `full`).

### Progress and reporting

- `--progress` / `--no-progress` – Show progress bars (default: enabled). Multi‑phase: indeterminate scanning, determinate processing.
- `--report` / `--no-report` – Print a detailed post‑run report with breakdown of skip/error causes.
- `--stats-only` – Already covered; often combined with `--report`.

### Redaction

- `--redact-secrets` / `--no-redact-secrets` – Redact common secret patterns from text content (best‑effort).

### Other

- `--version` – Show version and exit.
- `--log-level {info,warning,error}` – Logging verbosity (default: `info`).

## Output Format

The output is a single XML document.

Paths in output are always repository‑relative and use POSIX separators (`/`), even on Windows.

### Machine‑readable skip/error codes

When a file is skipped or fails processing, the XML includes machine‑readable attributes:

Skipped (intentional omission):

```xml
<file ... skipped="true" skip_code="text_size_limit">
```

Error (failed attempt):

```xml
<file ... skipped="true" error_code="text_read_error">
```

If available, a machine‑readable `<detail>` block is also emitted (deterministic JSON inside CDATA).

## Gitignore compatibility

`repo2xml` implements Git‑compatible `.gitignore` scoping:

- Each `.gitignore` applies to files under its directory.
- Patterns are matched against paths relative to the directory containing that `.gitignore`.
- The last matching pattern wins across all applicable `.gitignore` files.
- `.git` is not traversed by default (hard excluded), and ignore files inside `.git` are not considered.
- Additional hard‑excluded directories can be specified with `--hard-exclude`.

Note: compatibility with Git index (tracked files ignoring `.gitignore`) is intentionally out of scope.

## Fault Tolerance

`repo2xml` employs a "fail‑soft" strategy:

1. **Access Errors:** If a file cannot be read (permissions, locks), it is reported as an `<error>` in the output, but processing continues.
2. **Directory Access Errors:** If a directory cannot be listed (permissions, transient errors), it is skipped and a warning is logged.
3. **Scanner Entry Errors:** If a directory entry fails `is_file/stat/readlink`, it is skipped. A summary warning is logged after scanning (without spamming per file).

## Validate the XML

Quick check using Python's standard library:

```bash
python -c "import xml.etree.ElementTree as ET; ET.parse('context.xml'); print('XML OK')"
```

## Library Usage

Minimal example:

```python
from pathlib import Path
from repo2xml import Repo2XML, Repo2XMLConfig

engine = Repo2XML(Path("."), Repo2XMLConfig())
with open("context.xml", "wb") as f:
    stats = engine.export(f)

print(stats)
```
